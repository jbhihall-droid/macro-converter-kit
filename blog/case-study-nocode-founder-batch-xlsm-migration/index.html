From 3-Day Manual Export to 20 Minutes: How Priya Automated Her Client XLSM Migrations

Case study: No-code consultant, batch XLSM migration bottleneck, productised delivery

Priya runs a small automation consultancy helping professional services firms — accountants, insurance brokers, property managers — replace manual Excel workflows with modern cloud tools. Her typical engagement: audit the client's existing spreadsheets, identify what can move to Airtable, Notion, or Google Sheets, and build the replacement.

Most engagements go smoothly. But one category of client always created friction: firms that had built genuine business logic into XLSM workbooks. The macros weren't decoration — they were the system. Commission calculators, portfolio rebalancers, compliance checklists with automated flagging. Moving the data was easy. Moving the logic meant either rebuilding it from scratch in the new platform or preserving the Excel workflow as an adjunct system.

For adjunct-system clients, the migration handoff required converting 30–80 XLSM workbooks per engagement — with macros intact — to a format the client's new cloud environment could accept. This is where everything slowed down.

The Problem: A 3-Day Bottleneck at Every Engagement Handoff

"My process was embarrassing, honestly," Priya says. "I had a checklist: open each file in Excel, save as XLSB, rename, move to a staging folder, zip, upload. Repeat. For 60 files that was a full day, maybe two. And I'd still sometimes end up with a corrupted file or a macro that didn't survive the save-as."

She tried automating it with a Python script using xlwings, but the script required a locally installed Excel licence — which meant it only ran on her Windows machine, not in her cloud workspace. Clients on Mac couldn't run it at all.

She tried three online batch converters. Each one either:

  • Stripped the macros silently (discovered after upload when the client ran a routine and it failed)
  • Limited batch uploads to 5 files per session on the free tier
  • Required a paid plan at $49+/month for batch processing
"The expensive converter options were fine but I couldn't justify the cost for something I used 3-4 times a year. And the free ones kept eating the macros."

The bottleneck had a real cost. Priya tracked her hours: 18–22 hours per engagement spent on XLSM migration logistics. At her consulting rate, that was money she was absorbing or billing at cost — not value-adding work she could charge a premium for.

Frequently Asked Questions

Can batch XLSM conversion be done without a Windows machine? Reliably, with full macro preservation, no — not currently. The most reliable methods use COM automation via Excel on Windows. Cross-platform options either strip macros or require complex local LibreOffice configuration. For Mac-based workflows, the best current approach is running a Windows VM or cloud Windows instance for the conversion step.

What is the maximum number of XLSM files that can be batch converted reliably? With COM automation, batches of 200+ files have been tested successfully. Performance depends on file size and macro complexity; 60 standard business workbooks typically completes in 15–25 minutes on a mid-range Windows machine.

Does the COM automation approach require a paid Excel licence? Yes — it invokes the locally installed Excel application. A Microsoft 365 subscription (which most enterprise users already have) is sufficient. It does not require any additional Excel add-ins.

What happens to password-protected XLSM files in a batch migration? Password-protected files cannot be opened by automation without the password. They need to be handled separately — either manually or with a pre-processing step that logs which files are protected and queues them for manual review.

Is there a cloud-based alternative for teams without Windows access? The Macro-Safe Converter Launch Kit documents the current tool landscape including any cloud options. As of April 2026, no cloud-based tool provides reliable macro preservation for batch XLSM conversion — the kit explains why and what to use instead.
