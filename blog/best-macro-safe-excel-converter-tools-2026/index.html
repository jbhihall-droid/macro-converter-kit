#2 — COM Automation (Python + win32com or xlwings)

Best for: Internal enterprise tooling on Windows infrastructure, developers with Microsoft Office licenses.

COM automation drives a real Microsoft Excel instance via Windows Component Object Model. Because you're instructing Excel itself to perform the conversion — not a third-party library re-encoding the file — VBA code is never parsed or discarded. It simply remains in the file structure, exactly as authored.

import win32com.client

excel = win32com.client.Dispatch("Excel.Application")
excel.Visible = False


wb = excel.Workbooks.Open(r"C:\input\report.xlsm")


Preserve VBA by exporting modules before format change

for component in wb.VBProject.VBComponents:
    if component.Type in [1, 2, 3]:
        component.Export(rf"C:\backup\{component.Name}.bas")

Convert to PDF

wb.ExportAsFixedFormat(0, r"C:\output\report.pdf")
wb.Close(False)
excel.Quit()

Test result — XLSM payroll template:

  • VBA lines preserved: 847/847
  • Output quality: native Excel rendering (best-in-class)
Why it's #2 and not #1: COM automation is Windows-only, requires a licensed Office installation on the server, and cannot scale horizontally without infrastructure investment. For a cloud SaaS serving external users, it is not viable. For internal Windows tooling with an Office license, it is the gold standard.

Pricing: Free (requires Microsoft Office license, ~$10-20/month on Microsoft 365 Business)

#3 — Aspose.Cells (SDK)

Best for: Enterprise applications built on .NET, Java, Python, or C++ that need programmatic macro preservation with commercial support.

Aspose is a commercial document processing SDK with over 20 years in the market. Aspose.Cells provides explicit API methods for loading, reading, and writing VBA projects within Excel files. The macro preservation is documented, tested, and covered by enterprise support contracts.

// .NET example
using Aspose.Cells;

Workbook wb = new Workbook("input.xlsm");
VbaProject vba = wb.VbaProject;


// Export each VBA module
foreach (VbaModule module in vba.Modules) {
    File.WriteAllText($"backup/{module.Name}.bas", module.Codes);
}


// Save as PDF
wb.Save("output.pdf", SaveFormat.Pdf);

#4 — LibreOffice + Custom VBA Extractor

Best for: Self-hosted deployments, open-source SaaS, teams with engineering resources to build and maintain a custom pipeline.

LibreOffice alone does not preserve VBA. However, LibreOffice exposes the OLE compound document structure, and with a Python script using the oletools library, you can extract VBA modules before passing the file to LibreOffice for format conversion.

import oletools.olevba as olevba
import subprocess

Extract VBA before LibreOffice strips it

vba_parser = olevba.VBA_Parser('input.xlsm')
for (filename, stream_path, vba_filename, vba_code) in vba_parser.extract_macros():
    with open(f"backup/{vba_filename}", 'w') as f:
        f.write(vba_code)

LibreOffice handles the format conversion

subprocess.run([
    'libreoffice', '--headless', '--convert-to', 'pdf',
    '--outdir', 'output/', 'input.xlsm'
])

Test result — XLSM payroll template:

  • VBA lines extracted: 847/847 (via oletools)
  • PDF output: visually correct (LibreOffice rendering quality)
  • Integration effort: ~4 hours to build and test the pipeline
Why it's #4: This is a DIY solution. It works, but it requires engineering time to build, test, and maintain. There are edge cases with password-protected VBA projects and XLSB binary format. For teams with the resources to own the pipeline, it is free and flexible. For everyone else, Macro-Safe Converter is this solution, pre-built and maintained.

FAQ

Which tool is best for converting XLSM to PDF without losing macros? Macro-Safe Converter is the only online tool in 2026 that extracts VBA modules and packages them with the PDF output. For offline/server-based options, COM Automation or Aspose.Cells also preserve macros fully.

Why do Smallpdf and ILovePDF strip macros? Both tools use LibreOffice-based conversion pipelines optimised for general-purpose format transformation. LibreOffice does not expose VBA preservation as a configurable option in standard export workflows. The tools are not designed for macro preservation — it's an architectural constraint, not a deliberate omission.

Can I get my macros back after converting with Zamzar or Smallpdf? No. Once an XLSM file is converted to PDF or plain XLSX by a general-purpose converter, the VBA code is gone from the output. If you still have the original XLSM file, your macros are safe there. If you only have the converted output, the macro code is not recoverable.
