Excel Macro Backup Strategy: Protect Your VBA Before You Convert

Sponsored
By MacroKit  ·  Updated April 2026  ·  5 min read

Why Standard Backups Fail for Macros

Cloud auto-save (OneDrive, Dropbox) keeps file versions — but if you accidentally save over your XLSM with an XLSX, all versions after that point have no macros. The macro was lost the moment Excel stripped it, before the backup system saw the file.
The problem: If you save an XLSM as XLSX, and OneDrive syncs immediately, your backup also has no macros. File versioning saves the wrong thing.

Step 1: Export All VBA Modules as Text Files

VBA modules are pure text. Exporting them as .bas files means you can restore macros to any workbook, diff changes with git, and recover from format conversion accidents. 
# Requires xlwings (Windows + Excel installed)
import xlwings as xw
import os

wb = xw.Book('report.xlsm')
export_dir = 'vba_backup'
os.makedirs(export_dir, exist_ok=True)

for component in wb.api.VBProject.VBComponents:
    if component.Type in [1, 2, 3]:  # Module, Class, UserForm
        ext = {1: '.bas', 2: '.cls', 3: '.frm'}[component.Type]
        filename = f'{export_dir}/{component.Name}{ext}'
        component.Export(os.path.abspath(filename))
        print(f'Exported: {filename}')

print('All VBA modules backed up')

Step 2: Version Control the Exports with Git

# In your project directory:
git init vba-backup
cd vba-backup

# Copy your .bas exports here
cp ~/vba_backup/*.bas .

git add .
git commit -m 'Initial VBA backup — report.xlsm module exports'

# After any macro change:
# Re-export, then:
git add -A && git commit -m 'Update: added error handling to DataProcessor module'

Now you have a complete git history of every macro change. Recovering means copying a .bas file back into the VBE.

Step 3: Pre-Conversion Checklist

  1. Export all VBA modules to .bas files (script above)
  2. Commit to git or copy to a safe folder
  3. Record the macro inventory (module names + function count)
  4. Proceed with the format conversion
  5. If macros are missing after conversion, restore from .bas files

Quick Recovery: Reimporting .bas Files

# Restore a backed-up module into a workbook
import xlwings as xw

wb = xw.Book('recovered.xlsm')
# Import a .bas file back into VBE
wb.api.VBProject.VBComponents.Import(
    os.path.abspath('vba_backup/DataProcessor.bas')
)
print('Module restored')

Frequently Asked Questions

Does OneDrive or Dropbox back up Excel macros?

Yes — but with a critical caveat. They back up the file, including macros, as long as the file is saved as XLSM or XLSB. If you accidentally save as XLSX (stripping macros) and it syncs, that stripped version is what gets backed up.

Can I use git to version-control Excel VBA?

Yes, by exporting VBA modules to text (.bas files) using xlwings or the VBE File → Export. Text files are git-trackable. The XLSM binary itself is not meaningful in git diffs.

What is the best way to back up Excel macros before conversion?

Export all VBA modules as .bas files using xlwings before converting. Store them in a timestamped folder or commit to git. This gives you a recoverable copy that survives any format conversion.
